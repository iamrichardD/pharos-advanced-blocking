# User Guide

Welcome to the User Guide for **Pharos Advanced Blocking (pab)**. This guide will walk you through setting up credentials, running configuration commands, and using the interactive dashboard.

---

## Quickstart Guide

### Step 1: Install
Install the CLI using the install script or Debian package.

### Step 2: Navigate
Open your terminal and navigate to the directory where you have (or want to create) your Technitium `dnsApp.config` file.

### Step 3: Run `pab`
Run the `pab` command. `pab` automatically detects your local configuration files (like `dnsApp.config`), parses them, and saves modifications directly back to the disk. This local state management provides a secure offline safety net and GitOps integration.

### Configure Credentials
`pab` does NOT require a secrets file to start up. When you execute a command that communicates with your Technitium DNS server API (like `pab deploy`), `pab` will look for environment variables or a configuration file at `~/.config/pab/secrets.json` (protected with `chmod 600`). If none are found, it will safely prompt you for credentials or run in local dry-run validation mode.


You can also use the following environment variables:
- `PAB_BASE_URL`: The URL of your Technitium DNS server API.
- `PAB_API_TOKEN`: Your Technitium API access token.

---

## Mapping & Deploying

Once your configuration is ready, manage updates using the mapping and sync commands:

1. **Map Policies** (`pab map`): 
   Process your list data and map specific policies to devices or groups.
2. **Verify Changes** (`pab deploy --dry-run`): 
   Perform a dry-run check to validate configuration syntax and inspect changes without applying them.
3. **Sync Policies** (`pab deploy`): 
   Deploy your final configurations to Technitium. `pab` communicates securely via the Web API to apply modifications to Technitium instances with zero DNS service queries interruption.
